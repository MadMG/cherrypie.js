If you've come along here, you're probably searching for an easy and elegant way to convert some JSON data into a so called Rich Model and/or vice versa.

Congratulations! You've found what you've been looking for! :)

cherrypie.js is a rich-model populating model-factory which helps you to convert your incoming JSON object into a "rich" JS model object and vice versa with the help of a so called model-description, in no time!

With cherrypie.js you're now able to get primitive namespaced object values as well as computed properties which depend on a number of values from the received JSON, and endless nested child models (via child model-description(s)) as well as reduce/desolate the rich model object back to the object which is expected at the backend.

A short summary of what is possible with cherrypie.js:

  • Populate namespaced properties
    Declare a __namespace for the whole model and/or a namespaced value to be populated - shorthand variables when it comes to prepare your data!
  • Populate computed properties
    Each computed property (function) gets invoked with a prepared (namespace'd) origin and the cherrypie.js instance - fully flexible when it comes to prepare your data!
  • Populate nested child models
    Declare as many nested child models as you want within the __children property via a child model-description. Every nested child model-description is a full model-description on it's own - no limitations when it comes to prepare your data!
  • Desolate the rich model
    It is also possible to desolate the rich model into the JSON format you got by declaring all serializable data within the `__serializable` property, so you're able to convert it back and forth - no worries when it comes to prepare your data!

Take a look at the getting started page to see how easy it is to give your data a proper setup.

The cherrypie.js micro library is currently available in version 0.5.0 and is finalizing in it's definitions so that it'll be production ready in a blink of an eye - looking forward to release version 1.0.0 in June / July 2014.
